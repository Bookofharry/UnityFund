# Contributing to UnityFund

Thank you for your interest in contributing to UnityFund.

## Development Principles

- Follow the documented architecture and ADRs.
- Keep pull requests focused on a single feature or fix.
- Write clean, readable, and maintainable code.
- Add or update tests where appropriate.
- Update documentation when introducing architectural changes.

## Branch Strategy

- main — Production-ready code
- develop — Active development
- feature/<feature-name> — New features
- hotfix/<issue-name> — Critical fixes

## Commit Convention

Examples:

feat: add contribution service

fix: prevent duplicate webhook processing

docs: update fund rules API

refactor: simplify payout workflow

test: add payment integration tests

## Pull Requests

Every pull request should:

- Reference the related issue (if applicable).
- Pass all automated tests.
- Follow project coding standards.
- Include documentation updates where necessary.

## Code Reviews

Code should be reviewed before merging into the main branch.

Constructive feedback is encouraged. Focus on improving the code, not criticizing the author.
